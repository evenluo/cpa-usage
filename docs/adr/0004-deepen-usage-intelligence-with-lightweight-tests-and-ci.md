# Deepen Usage Intelligence with lightweight tests and CI

CPA Usage will deepen the existing Usage Intelligence path without adding a new backend architecture package: analytics read model types stay in `internal/repository`, analytics implementation is split by same-package Go files, `internal/api` remains the HTTP contract owner, and duplicated service-layer analytics DTOs are removed. The frontend will introduce `web/src/features/usage-intelligence` and `web/src/features/reference-data` for tested view-model and interaction logic while route files keep state, data fetching, and layout composition. Tests stay lightweight and idiomatic: Go uses standard `testing` with `t.Run`, frontend uses Vitest and Testing Library, and GitHub Actions runs the existing Makefile verification path; this intentionally avoids `internal/analytics`, `goconvey`, OpenAPI/codegen, release automation, coverage upload, and `golangci-lint` for this change.
